# Radar Pro — Подробный План Разработки (Roadmap)

**Версия:** 1.0 (Черновик)  
**Дата:** 2026-07-16  
**Автор:** code_warlord

Этот документ содержит **полное поэтапное разбиение** разработки Radar Pro — от создания каркаса проекта до полной реализации всей архитектуры.

---

## Фаза 0: Фундамент и Настройка Проекта (1–2 недели)

**Цель:** Создать профессиональный монопо, настроить все инструменты и инфраструктуру.

### Этап 0.1: Инициализация Репозитория
- Создать Turborepo (или Nx)
- Приложения: `web`, `worker`, `bff`
- Пакеты: `core`, `engine`, `connectors`, `database`, `manifests`, `ui-components`
- Настроить TypeScript (strict), ESLint, Prettier, Husky
- Vitest + React Testing Library
- `.env.example`, README.md

**Результат:** Рабочая структура монопо  
**Релиз:** `v0.1.0` — Скелет проекта

### Этап 0.2: Настройка Cloudflare
- Wrangler
- Создание D1 + базовая схема (`datasets`, `repositories`)
- R2 (Journal), KV (кэш)
- Миграции (Drizzle или SQL)
- CI/CD для Pages + Workers

**Результат:** Локальная и preview инфраструктура  
**Релиз:** `v0.1.1` — Инфраструктура готова

### Этап 0.3: State Management и Основы Frontend
- React 19 + Vite + TypeScript
- TanStack Query, **Zustand**, nuqs, React Router, Zod
- Базовая стратегия управления состоянием
- Стартер для UI-компонентов

**Результат:** Базовые Zustand-хранилища  
**Релиз:** `v0.1.2` — Основы Frontend

## Фаза 1: GitHub Radar MVP (3–5 недель)

**Цель:** Полноценный минимально жизнеспособный продукт (только GitHub).

### Этап 1.1: Слой Данных и Коннекторы
- GitHub Connector
- Journal в R2
- Entity Store в D1
- Нормализация данных

**Релиз:** `v0.2.0` — Сбор данных

### Этап 1.2: Feature Engine
- Расчёт базовых признаков (`stars_velocity_7d`, `release_frequency` и др.)
- Простой исполнитель

**Релиз:** `v0.2.1` — Features

### Этап 1.3: Metric Engine + Простой DAG
- Первые метрики (`health_score_v1`, `growth_score_v1`)
- **Внедрение базового DAG Runtime**
- Версионирование метрик

**Релиз:** `v0.3.0` — Базовый скоринг

### Этап 1.4: BFF и API
- Hono Worker
- Query API (`/repositories/:id`, `/radar/trending`)
- Command API

**Релиз:** `v0.3.1` — API Layer

### Этап 1.5: React Frontend
- Дашборд, карточки репозиториев
- Детальная страница проекта
- Поиск и фильтры
- Zustand для клиентского состояния

**Релиз:** `v0.4.0` — GitHub Radar MVP

### Этап 1.6: Тестирование и Деплой
- Тесты, мониторинг, production deploy

**Релиз:** `v0.4.1` — MVP в продакшене
