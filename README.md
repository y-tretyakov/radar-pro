> *GitHub уже давно перестал быть просто хранилищем кода. Это огромная экосистема, в которой ежедневно рождаются тысячи новых проектов. Но среди миллионов репозиториев лишь единицы становятся новыми Docker, Neovim, Bun, Axum или Ollama. Главный вопрос — как найти их раньше остальных?*

Именно эту задачу решает **Radar Pro**.

Это не очередной клиент для GitHub и не красивая оболочка над GitHub Search API. Это аналитическая платформа, цель которой — обнаруживать перспективные Open Source проекты ещё до того, как они окажутся в трендах.

---

# Почему появился Radar Pro

Практически каждый разработчик сталкивался с одной и той же проблемой.

Допустим, вы решили найти новую библиотеку на Rust, современный TUI-фреймворк или перспективный AI-инструмент.

Открываете GitHub.

Пишете запрос.

Получаете тысячи результатов.

Дальше начинается ручная работа:

* сортировка по stars;
* просмотр README;
* изучение активности;
* анализ коммитов;
* проверка релизов;
* сравнение нескольких проектов.

Через полчаса становится понятно, что GitHub отлично отвечает на вопрос:

> **"Что уже существует?"**

Но совершенно не отвечает на другой, куда более интересный вопрос:

> **"Что станет популярным через полгода?"**

Radar Pro создавался именно для ответа на этот вопрос.

---

# Не поиск. Радар.

Ключевая идея проекта — отказаться от мышления "поисковой системы".

Поиск — это всего лишь один из инструментов.

Главная задача Radar Pro — обнаружение сигналов.

Представьте настоящий радар.

Он не показывает абсолютно всё вокруг.

Он выделяет именно те объекты, которые заслуживают внимания.

Radar Pro работает аналогично.

Вместо миллионов репозиториев пользователь получает подборку проектов, действительно имеющих потенциал роста.

---

# Что анализирует Radar Pro

Каждый репозиторий проходит через несколько независимых этапов анализа.

Каждый этап отвечает за определённую характеристику проекта.

В результате формируется целостная картина.

---

# Growth

Первый и самый очевидный показатель.

Количество stars само по себе практически ничего не говорит.

Проект со 100 000 stars вполне может оказаться заброшенным.

И наоборот — репозиторий со 300 stars способен за месяц превратиться в новый тренд.

Поэтому анализируется не абсолютное количество звёзд, а скорость их появления.

Например:

```
Stars: 250

Через неделю:

Stars: 620
```

Рост составил:

```
+370 stars
```

Именно скорость изменения становится первым важным сигналом.

---

# Velocity

Radar Pro вычисляет скорость роста.

Условно:

$$\text{Star Velocity} = \frac{\text{New Stars}}{\text{Days}}$$

Например:

```
210 stars

за 7 дней

=

30 stars/day
```

Это позволяет сравнивать проекты разных размеров.

---

# Acceleration

Но даже высокая скорость не всегда означает успех.

Куда интереснее понять:

**растёт ли скорость сама по себе?**

Например:

```
Неделя №1

+40 stars
```

```
Неделя №2

+120 stars
```

```
Неделя №3

+480 stars
```

Получается ускорение.

Формула выглядит достаточно просто:

$$\text{Acceleration} = \frac{\text{Current Growth}}{\text{Previous Growth}}$$

Если коэффициент значительно превышает единицу, это говорит о появлении вирусного эффекта.

Именно такие проекты получают статус:

* Rising Star
* Exploding

---

# Momentum

Следующий уровень анализа — активность разработки.

Даже стремительно растущий проект может оказаться "одноразовым".

Поэтому Radar Pro анализирует:

* количество коммитов;
* частоту релизов;
* регулярность обновлений;
* непрерывность разработки.

Например:

```
Последний commit

4 часа назад
```

значительно ценнее, чем

```
Последний commit

11 месяцев назад
```

Даже если stars у второго проекта заметно больше.

---

# Community

Open Source живёт сообществом.

Поэтому Radar Pro анализирует не только автора проекта.

Важнее другое.

Сколько людей действительно участвует в разработке?

Учитываются:

* contributors;
* forks;
* pull requests;
* внешние контрибьюторы;
* активность обсуждений.

Большое количество независимых участников — сильный сигнал того, что проект становится частью экосистемы.

---

# Quality

Популярность ещё не означает качество.

Radar Pro пытается оценить зрелость проекта.

Проверяется наличие:

* README;
* LICENSE;
* CONTRIBUTING;
* CHANGELOG;
* тестов;
* документации;
* GitHub Actions;
* примеров использования.

Если репозиторий выглядит аккуратно и поддерживается по всем правилам Open Source — его рейтинг повышается.

---

# Trend

Отдельный модуль оценивает технологический контекст.

Сегодня далеко не все технологии развиваются одинаково быстро.

Некоторые направления получают значительно больше внимания сообщества.

Например:

* AI
* Rust
* WASM
* Kubernetes
* eBPF
* MCP
* Terminal UI
* Developer Tools

Если проект развивается внутри быстрорастущей экосистемы, это становится дополнительным положительным сигналом.

Важно понимать:

Radar Pro **не штрафует** проекты на Java, C# или Python.

Технология сама по себе не влияет на рейтинг.

Она лишь может немного усилить положительные сигналы.

---

# Radar Score

Все показатели объединяются в единую оценку.

Каждый компонент имеет собственный вес.

```
Growth      25%

Momentum    20%

Community   20%

Quality     20%

Trend       15%
```

После нормализации получается итоговая оценка:

```
Radar Score

0…100
```

Важно подчеркнуть:

Radar Score — это не оценка качества кода.

И не прогноз будущего.

Это вероятность того, что проект заслуживает внимания именно сейчас.

# Почему проект оказался в выдаче

Одна из самых необычных особенностей Radar Pro — объяснимость.

Большинство современных рекомендаций выглядят так:

> "Алгоритм решил."

Почему?

Непонятно.

Radar Pro идёт противоположным путём.

Каждая рекомендация сопровождается набором сигналов.

Например:

```
Why Interesting

🔥 Высокая скорость роста

🔥 Активная разработка

🔥 Хорошая документация

🔥 Растущее сообщество

🔥 Популярная технологическая область
```

Пользователь видит не только итоговую оценку.

Он понимает, **из чего она сложилась**.

# Почему без искусственного интеллекта

На первый взгляд может показаться странным, что в проекте отсутствует LLM.

Это сознательное решение.

Вместо генерации красивых текстов Radar Pro использует собственный движок правил.

Каждое объяснение строится на основании конкретных метрик.

Такой подход имеет несколько преимуществ.

Во-первых, он полностью прозрачен.

Во-вторых, работает офлайн.

В-третьих, идеально подходит для статического размещения на GitHub Pages.

И наконец — пользователь всегда может проследить происхождение каждой рекомендации.

Никакой "магии".

Только данные.

---

# Не GitHub-клиент

Radar Pro не пытается заменить GitHub.

Наоборот.

GitHub остаётся местом разработки.

Radar Pro становится местом исследования.

Это своеобразный технический радар, который помогает увидеть перспективные проекты ещё до того, как о них начнут говорить все.

# Планы развития

Текущая версия ориентирована на анализ репозиториев в реальном времени.

В дальнейшем проект планируется развивать в сторону полноценной аналитической платформы.

Среди возможных направлений:

* история изменения Radar Score;
* графики роста;
* сравнение нескольких репозиториев;
* отслеживание проектов;
* персональные подборки;
* уведомления о резком росте;
* анализ конкурентов;
* поиск альтернатив;
* AI-аналитик, объясняющий причины роста проекта на основе уже рассчитанных метрик.

При этом базовая философия останется неизменной.

Сначала — объективные данные.

Потом — аналитика.

И только после этого — интеллектуальные рекомендации.

# Вместо заключения

Open Source развивается невероятно быстро. Каждый день появляются сотни новых библиотек, инструментов и фреймворков. Большинство из них так и остаются незамеченными. Некоторые меняют целые отрасли.

Самая сложная задача — не найти уже популярный проект. Самая сложная задача — заметить его тогда, когда о нём ещё знают лишь несколько сотен человек.

Именно для этого создаётся **Radar Pro**.

Не как ещё один поиск по GitHub.

А как инструмент, который помогает увидеть будущее Open Source немного раньше остальных.

# Development

Monorepo (pnpm + Turborepo). Requires **Node.js 22+** and **pnpm 9+**.

```bash
pnpm install

# Quality gates
pnpm typecheck
pnpm lint
pnpm test
pnpm build

# Local dev (all apps via Turbo)
pnpm dev

# Or per-app:
pnpm --filter @radar-pro/web dev      # http://localhost:5173
pnpm --filter @radar-pro/api dev      # http://localhost:8787
pnpm --filter @radar-pro/worker dev
```

See [docs/DEVELOPMENT.md](docs/DEVELOPMENT.md) for structure and environment notes.
